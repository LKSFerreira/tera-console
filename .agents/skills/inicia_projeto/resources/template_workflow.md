# Fluxo de Trabalho Obrigatório

1. Analisar o contexto antes de alterar arquivos.
2. Discutir a proposta técnica com o usuário quando a mudança não for explicitamente imperativa.
3. Refinar a solução com base no feedback recebido.
4. Executar somente o que foi aprovado.
5. Validar o resultado antes de encerrar a tarefa.

## Regra de Ouro

- Proibido implementar sem entender o contexto.
- Sempre alinhar decisões relevantes antes de mudanças estruturais.
